# RPG 玩法改進追蹤計畫

## Summary

目標是把目前「探索 -> 隨機戰鬥 -> 攻擊/藥水/逃跑」的循環，升級成有戰術選擇、怪物差異、章節 Boss 和可累積成長的玩法。

這份檔案使用 checkbox 格式追蹤進度。完成任何項目後，直接把 `- [ ]` 改成 `- [x]`。

## Key Changes

- [x] **技能系統**
  - 新增玩家技能：普通攻擊、重擊、防禦、詛咒/燃燒。
  - 戰鬥 UI 顯示技能按鈕、技能描述、不可用狀態。
  - 技能需配合現有怪物 `attack / hurt / death` 動畫節奏。

- [x] **怪物特殊能力**
  - 史萊姆：低攻擊，偶爾回血。
  - 骷髏兵：高防禦。
  - 野狼：可能連擊。
  - 暗影獸：提高逃跑失敗懲罰。
  - 黑燭信徒：施加持續傷害或降低玩家防禦。
  - 其他怪物保留基礎能力，但數值依章節提升。

- [x] **Boss 關卡節點**
  - 每個場景進度達標後觸發 Boss，而不是直接進入下一區。
  - Boss 擁有固定名稱、較高生命值、特殊技能和獎勵。
  - 擊敗 Boss 後才解鎖下一個場景。

- [x] **探索事件選擇**
  - 探索時加入事件選項，例如祭壇、寶箱、岔路、受傷旅人。
  - 每個事件提供 2-3 個選擇：安全、風險、消耗資源。
  - 結果可影響 HP、金幣、藥水、戰鬥觸發、故事進度。

- [x] **遺物系統**
  - 新增少量被動遺物，作為探索或 Boss 獎勵。
  - 遺物效果先保持簡單：擊殺回血、提高藥水效果、降低逃跑懲罰、首次死亡保護金幣。
  - 個人資訊或背包頁需顯示已獲得遺物。

## Interfaces And Data

- [x] **玩家狀態欄位**
  - `skills`: 已解鎖技能清單。
  - `effects`: 當前戰鬥狀態效果。
  - `relics`: 已獲得遺物清單。

- [x] **怪物資料欄位**
  - `ability`: 怪物特殊能力 ID。
  - `boss`: 是否為 Boss。
  - `statusEffects`: 怪物身上的持續效果。

- [x] **戰鬥流程**
  - 玩家選擇技能。
  - 套用技能傷害或效果。
  - 若怪物未死亡，執行怪物能力或普通攻擊。
  - 結算持續效果。
  - 更新 UI、日誌、動畫。

## Assumptions

- [x] 第一版不做複雜職業、裝備欄或存檔系統。
- [x] 優先改進戰鬥決策密度，再擴展長期養成。
- [x] 使用現有單檔 `index.html` 架構，除非功能拆分已明顯降低維護成本。
- [x] 新增玩法需維持中英文介面文案完整。

## Next Phase Plan

下一階段目標是在已完成的技能、怪物能力、Boss、探索事件、遺物系統之上，補上更穩定的長期遊玩循環與品質驗證。

## Next Key Changes

- [x] **存檔與讀檔**
  - 使用 `localStorage` 保存玩家等級、HP、金幣、藥劑、技能、遺物、劇情進度、已擊敗 Boss。
  - 提供「新遊戲 / 繼續遊戲 / 重置進度」入口。
  - 存檔版本需可遷移，避免之後新增欄位造成舊存檔壞掉。

- [x] **戰鬥與獎勵平衡**
  - 調整普通怪、強敵、Boss 的 HP / attack / defense 成長曲線。
  - 檢查技能傷害、燃燒、重擊、防禦與怪物能力是否有明顯最優解。
  - 讓 Boss 與探索事件的金幣、經驗、遺物節奏更穩定。

- [x] **遺物與探索擴充**
  - 新增 2-3 個低複雜度遺物，例如低血量加防、首次進入 Boss 戰回血、探索事件額外金幣。
  - 新增至少 2 個探索事件，避免中後期重複感過高。
  - 補上遺物取得來源提示，讓玩家知道哪些 Boss 或事件可能掉落遺物。

- [x] **冒險狀態與結局回饋**
  - 最終 Boss 擊敗後顯示明確的結局或 expedition complete 狀態。
  - 冒險頁顯示目前章節目標，例如「推進 3 次後將遇到 Boss」。
  - Boss pending 時顯示更清楚的阻擋提示與挑戰入口。

- [x] **UI 與可讀性整理**
  - 整理戰鬥 log 與一般 log 的訊息優先順序，避免重要事件被洗掉。
  - 背包與個人資訊頁增加更清楚的遺物分區與狀態標記。
  - 檢查手機版戰鬥、探索事件、遺物卡片是否仍無重疊或截斷。
