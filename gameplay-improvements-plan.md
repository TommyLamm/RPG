# RPG 玩法改進追蹤計畫

## Summary

目標是把目前「探索 -> 隨機戰鬥 -> 攻擊/藥水/逃跑」的循環，升級成有戰術選擇、怪物差異、章節 Boss 和可累積成長的玩法。

這份檔案使用 checkbox 格式追蹤進度。完成任何項目後，直接把 `- [ ]` 改成 `- [x]`。

## Key Changes

- [x] **技能系統**
  - 新增玩家技能：普通攻擊、重擊、防禦、詛咒/燃燒。
  - 戰鬥 UI 顯示技能按鈕、技能描述、不可用狀態。
  - 技能需配合現有怪物 `attack / hurt / death` 動畫節奏。

- [ ] **怪物特殊能力**
  - 史萊姆：低攻擊，偶爾回血。
  - 骷髏兵：高防禦。
  - 野狼：可能連擊。
  - 暗影獸：提高逃跑失敗懲罰。
  - 黑燭信徒：施加持續傷害或降低玩家防禦。
  - 其他怪物保留基礎能力，但數值依章節提升。

- [ ] **Boss 關卡節點**
  - 每個場景進度達標後觸發 Boss，而不是直接進入下一區。
  - Boss 擁有固定名稱、較高生命值、特殊技能和獎勵。
  - 擊敗 Boss 後才解鎖下一個場景。

- [ ] **探索事件選擇**
  - 探索時加入事件選項，例如祭壇、寶箱、岔路、受傷旅人。
  - 每個事件提供 2-3 個選擇：安全、風險、消耗資源。
  - 結果可影響 HP、金幣、藥水、戰鬥觸發、故事進度。

- [ ] **遺物系統**
  - 新增少量被動遺物，作為探索或 Boss 獎勵。
  - 遺物效果先保持簡單：擊殺回血、提高藥水效果、降低逃跑懲罰、首次死亡保護金幣。
  - 個人資訊或背包頁需顯示已獲得遺物。

## Interfaces And Data

- [ ] **玩家狀態欄位**
  - `skills`: 已解鎖技能清單。
  - `effects`: 當前戰鬥狀態效果。
  - `relics`: 已獲得遺物清單。

- [ ] **怪物資料欄位**
  - `ability`: 怪物特殊能力 ID。
  - `boss`: 是否為 Boss。
  - `statusEffects`: 怪物身上的持續效果。

- [ ] **戰鬥流程**
  - 玩家選擇技能。
  - 套用技能傷害或效果。
  - 若怪物未死亡，執行怪物能力或普通攻擊。
  - 結算持續效果。
  - 更新 UI、日誌、動畫。

## Test Plan

- [ ] 更新 `tests/browser-smoke.js`
  - 驗證技能按鈕存在並可觸發。
  - 驗證防禦技能會降低下一次受到的傷害。
  - 驗證持續傷害能在回合結算時生效。
  - 驗證怪物特殊能力至少各有一個可觀測結果。
  - 驗證 Boss 擊敗後才推進場景。

- [ ] 更新 `tests/verify-assets.js`
  - 若新增 Boss 或 UI 素材，確認 manifest 有列入並且檔案存在。

- [ ] 手動驗收
  - 從新遊戲進入探索。
  - 遇到普通怪物並使用至少兩種技能。
  - 觸發一個探索選擇事件。
  - 達成場景進度並進入 Boss 戰。
  - 擊敗 Boss 後解鎖下一區。
  - 確認中英文 UI 文字沒有缺漏。

## Assumptions

- [ ] 第一版不做複雜職業、裝備欄或存檔系統。
- [ ] 優先改進戰鬥決策密度，再擴展長期養成。
- [ ] 使用現有單檔 `index.html` 架構，除非功能拆分已明顯降低維護成本。
- [ ] 新增玩法需維持中英文介面文案完整。
